## 1. プロジェクト概要

### 1.1 背景・目的

パソコン教室「ほほえみラボ」のシニア生徒さんが、日常の体験・気づき・想いを気軽に記録・共有できる場を提供する。

主な目的は以下の3つ。

1. **スマホ操作の自然な練習**: 写真撮影・文字入力・投稿を「楽しみながら」練習できる機会の創出
2. **教室内コミュニケーションの活性化**: 「ここ行ってみたい」「美味しそう」をきっかけにした、リアルな会話の促進
3. **地域ナレッジの蓄積（運営視点・非公開）**: 将来的なAIチャットボット開発に向け、AIが持っていない南信州のリアルな口コミ・スポット・体験情報をフェーズを通じて蓄積する

### 1.2 プロジェクト名について

| 項目                 | 内容                                                                    |
| :------------------- | :---------------------------------------------------------------------- |
| **サービス名（案）** | **Minapedia（ミナペディア）**                                           |
| **由来**             | 南信（Mina）+ Log（記録）。地域色を前面に出しつつ、スタイリッシュな印象 |
| **URL（初期）**      | `https://minapedia.vercel.app/`                                         |
| **ドメイン（将来）** | 未定。独自ドメイン取得を推奨（例: `minapedia.jp`）                      |

### 1.3 コンセプト

**「写真がなくても、投稿できる。南信州の日常をみんなでのこす場所。」**

Instagramのような写真中心の楽しさを取り入れつつ、以下を重視する。

- 写真がなくても見栄えのよいカードUI（X/Twitterライクなテキスト投稿も美しく）
- 情報過多・SNS疲れを防ぐクローズドな設計
- 評価・比較されない安心感（いいね数の非表示、コメントなし）
- シニアが迷わない「引き算のUI」

### 1.4 ターゲットユーザー

- ほほえみラボの生徒さん（主にシニア層、60〜80代）
- スマホ操作に少し不安がある方、InstagramなどのSNSは難しいと感じている方
- 想定ユーザー数: **現状20名 → 最大30名程度（MVP運用期）**

## 3. UI/UX 基本方針

### 3.1 最重要原則：「写真がなくても美しいカード」

写真あり・なし、両方のパターンで見栄えのよいUIを実現する。

**写真ありの投稿（Instagram風）**

- 写真を上部に大きく表示
- その下にニックネーム・タイトル・テキストを配置

**写真なしの投稿（X/Twitter風）**

- テーマに対応したイラスト・アイコンをカード上部に自動表示（例：「行ってみたいところ」→ 地図アイコン＋温かみのあるカラー背景）
- テキスト（タイトル・感想）を大きく・読みやすく表示
- エリアタグ・テーマバッジを視覚的に強調

### 3.2 積極的に取り入れる要素

- 写真中心 or テキスト中心、どちらでも見栄えのよい投稿カード
- ニックネーム表示（本名非公開による安心感）
- シンプルなリアクション機能
- スマホ操作しやすい下部固定ナビゲーション（3項目）
- 新着順タイムライン

### 3.3 排除・弱める要素（シニア配慮）

- 無限スクロール → 「もっと見る」ボタンで区切る
- 動画・リールコンテンツ
- コメント欄（トラブル防止、返信プレッシャーの排除）
- フォロー機能、複雑なプロフィール設定
- いいね数・リアクション数の競争的表示（数は非表示 or 控えめに）
- 情報が流れすぎるUI

### 3.4 アクセシビリティ配慮

- フォントサイズ: 最小16px、本文18px以上推奨
- 高コントラスト配色
- ボタンのタップ領域: 最小48×48px
- エラーメッセージは日本語で平易に表示

## 4. 機能要件（Phase 1 MVP）

### 4.1 生徒さん向け機能

| 機能                         | 詳細                                                                                                                   |
| :--------------------------- | :--------------------------------------------------------------------------------------------------------------------- |
| **アカウント登録・ログイン** | 教室で先生と一緒に初回登録。メールアドレス＋パスワード。ニックネームで参加。ログイン状態は長期保持（セッション延長）。 |
| **投稿作成**                 | 写真（0〜3枚・任意）と型にはめたフォームで投稿。写真なしでも投稿可能。                                                 |
| **投稿編集・削除**           | 自分の投稿はいつでも修正・削除可能。                                                                                   |
| **一覧閲覧**                 | 「みんなの新着」「テーマ別」「自分の投稿」の3タブ。                                                                    |
| **リアクション**             | 「すてき」「行ってみたい」「参考になった」の3種ボタンのみ。数は控えめ表示。                                            |
| **投稿詳細閲覧**             | カードから詳細画面へ遷移。写真を大きく表示、全文表示。                                                                 |

### 4.2 管理者（先生）向け機能

| 機能                     | 詳細                                                                             |
| :----------------------- | :------------------------------------------------------------------------------- |
| **投稿承認**             | 生徒の投稿は初期状態「確認中」。管理画面から「公開」に変更することで表示される。 |
| **差し戻し・非表示**     | 個人情報記載など不適切な内容の場合、差し戻しまたは非表示。                       |
| **おすすめ固定表示**     | ホーム画面上部に「今週のおすすめ投稿」を最大3件まで固定表示。                    |
| **管理者アカウント管理** | 管理者は1名（将来2名まで招待可能な設計）。Supabase Authのロール管理を使用。      |

## 5. 投稿フォーム詳細仕様

### 5.1 入力項目

生徒さんが迷わず入力できるよう、型にはめたフォーム設計。

| #   | 項目名               | 入力方式                   | 必須 | 備考                                       |
| :-- | :------------------- | :------------------------- | :--- | :----------------------------------------- |
| 1   | **写真**             | 画像アップロード（0〜3枚） | 任意 | なしでも投稿可。クライアント側で圧縮処理   |
| 2   | **テーマ**           | 選択（1つ）                | 必須 | 選択肢は下記参照                           |
| 3   | **タイトル**         | テキスト入力               | 必須 | 例：天龍峡のカフェに行ってきました         |
| 4   | **場所の名前**       | テキスト入力               | 推奨 | 例：〇〇喫茶店（ナレッジDB化の核心データ） |
| 5   | **エリア**           | ドロップダウン             | 推奨 | 飯田市 / 高森町 / 阿智村 / 松川町 / 其他   |
| 6   | **どこがよかった？** | テキストエリア             | 任意 | 例：歩きやすかった、ケーキが美味しかった   |
| 7   | **ひとこと感想**     | テキストエリア             | 任意 | 例：また春にも行ってみたいです             |

> **送信ボタンのテキスト**: 「先生に送る（投稿する）」→ 心理的ハードルを下げる

### 5.2 テーマ選択肢（Phase 1）

**行ってきた系（写真あり想定）**

- 🗺 よかった場所
- 🍽 おいしいもの
- 🌸 きれいな景色
- 🔄 また行きたい

**やりたい系（写真なし想定 ＝ Phase 2準備）**

- 📍 行ってみたいところ
- ✨ やってみたいこと

> 「行ってみたいところ」「やってみたいこと」は、写真なしでもデフォルトで映えるカードデザインを適用する。生徒さんには「夢や希望を書いてみましょう」という形で案内。

### 5.3 写真なし時のカードデザイン対応表

| テーマ             | 自動表示アイコン      | 背景カラー         |
| :----------------- | :-------------------- | :----------------- |
| よかった場所       | 🗺 地図アイコン       | ナチュラルグリーン |
| おいしいもの       | 🍽 フォークアイコン   | ウォームオレンジ   |
| きれいな景色       | 🌸 花アイコン         | ソフトピンク       |
| また行きたい       | ⭐ スターアイコン     | ゴールドイエロー   |
| 行ってみたいところ | 📍 ピンアイコン       | スカイブルー       |
| やってみたいこと   | ✨ スパークルアイコン | ラベンダー         |

## 6. 画面設計

### 6.1 画面一覧とナビゲーション

モバイルファースト設計。画面下部に3つの固定ナビゲーションを配置。

| #   | ナビ名        | 画面           |
| :-- | :------------ | :------------- |
| 1   | 🏠 ホーム     | みんなの新着   |
| 2   | ✏️ 投稿する   | 投稿フォーム   |
| 3   | 👤 マイページ | 自分の投稿一覧 |

### 6.2 ホーム画面（みんなの新着）

上から順に以下を配置。

1. **安心メッセージ（初回のみ表示）**: 「このサービスは教室の生徒さんだけが見られます」
2. **先生おすすめ**: 管理者がピックアップした投稿（最大3件・固定表示）
3. **テーマで絞る**: テーマボタンで絞り込み
4. **みんなの新着**: 新着順に5件ずつ表示、「もっと見る」ボタンで追加取得

### 6.3 投稿カードの表示要素

```
┌─────────────────────────────┐
│ [写真 or テーマアイコン＋背景]   │
├─────────────────────────────┤
│ [テーマバッジ] [エリアタグ]      │
│ タイトル（大きく・太字）         │
│ ニックネーム                   │
│ 本文（2行プレビュー）           │
│ [続きを読む]                   │
├─────────────────────────────┤
│ すてき 👍  行ってみたい 📍  参考になった 💡 │
└─────────────────────────────┘
```

### 6.4 投稿詳細画面

- 写真（最大3枚・スワイプ切り替え）or テーマアイコン＋背景
- タイトル・ニックネーム・エリア・テーマ
- 全文表示（どこがよかった？ ＋ ひとこと感想）
- リアクションボタン
- 場所の名前（将来的にGoogle Maps連携予定）
- 自分の投稿の場合のみ「編集・削除」ボタン表示

### 7.1 技術スタック

### 7.2 主要テーブル定義

#### `profiles`（ユーザー情報）

| カラム       | 型        | 説明                                 |
| :----------- | :-------- | :----------------------------------- |
| `id`         | uuid (PK) | Supabase Auth の user_id             |
| `nickname`   | text      | ニックネーム（表示名）               |
| `role`       | text      | `student` / `admin`                  |
| `age_range`  | text      | 年代（60代・70代・80代 etc）任意入力 |
| `gender`     | text      | 性別（任意・非公開）                 |
| `created_at` | timestamp | 登録日時                             |

#### `posts`（投稿データ）

| カラム        | 型        | 説明                               |
| :------------ | :-------- | :--------------------------------- |
| `id`          | uuid (PK) |                                    |
| `user_id`     | uuid (FK) | profiles.id                        |
| `category_id` | int (FK)  | categories.id                      |
| `title`       | text      | タイトル                           |
| `spot_name`   | text      | 場所の名前（ナレッジDB化の核心）   |
| `area`        | text      | エリア（飯田市 etc）               |
| `body_good`   | text      | どこがよかった？                   |
| `body_memo`   | text      | ひとこと感想                       |
| `status`      | text      | `pending` / `published` / `hidden` |
| `is_featured` | boolean   | 先生おすすめ固定フラグ             |
| `created_at`  | timestamp |                                    |
| `updated_at`  | timestamp |                                    |

#### `post_images`（投稿画像）

| カラム         | 型        | 説明                    |
| :------------- | :-------- | :---------------------- |
| `id`           | uuid (PK) |                         |
| `post_id`      | uuid (FK) | posts.id                |
| `storage_path` | text      | Supabase Storage のパス |
| `order_index`  | int       | 表示順                  |

#### `categories`（テーマ）

| カラム  | 型       | 説明                                   |
| :------ | :------- | :------------------------------------- |
| `id`    | int (PK) |                                        |
| `name`  | text     | よかった場所 etc                       |
| `icon`  | text     | 絵文字アイコン                         |
| `color` | text     | デフォルト背景カラー（HEX）            |

#### `reactions`（リアクション）

| カラム       | 型        | 説明                          |
| :----------- | :-------- | :---------------------------- |
| `id`         | uuid (PK) |                               |
| `post_id`    | uuid (FK) | posts.id                      |
| `user_id`    | uuid (FK) | profiles.id                   |
| `type`       | text      | `suteki` / `ikitai` / `sanko` |
| `created_at` | timestamp |                               |

> UNIQUE制約: `(post_id, user_id, type)` — 1ユーザー1投稿につき各タイプ1回

### 7.3 Supabase 利用規模・プラン

| 項目               | 想定値                                     |
| :----------------- | :----------------------------------------- |
| 想定ユーザー数     | 最大30名                                   |
| 月間投稿数（想定） | 30〜100件                                  |
| ストレージ使用量   | 画像圧縮後 平均500KB/投稿 → 月最大50MB程度 |
| **プラン**         | **Free プラン（MVP期間は十分）**           |

> Free プランは 500MB DB・1GB Storage・50,000 MAU まで無料。30名規模では問題なし。  
> プロジェクトの自動停止（7日間非アクティブ）に注意 → Supabase Keeper等で対策済みの構成を採用。

## 8. セキュリティ・運用方針

### 8.1 安心して使えるメッセージ（画面上に明示）

| 表示場所         | メッセージ                                       |
| :--------------- | :----------------------------------------------- |
| ホーム上部       | 「このサービスは教室の生徒さんだけが見られます」 |
| 投稿フォーム上部 | 「投稿は先生が確認してから表示されます」         |
| プロフィール設定 | 「本名や大切な個人情報は表示されません」         |
| 投稿詳細         | 「あとで自分で編集・削除できます」               |

### 8.2 技術的セキュリティ対策

- Supabase RLS（Row Level Security）による認証ユーザーのみアクセス制御
- 画像アップロード: クライアント側で圧縮・容量エラー防止
- ログインセッション: 長期保持（30日以上）でパスワード忘れ対策
- 管理者ロールは `profiles.role = 'admin'` で管理

### 8.3 個人情報の取り扱い

- ニックネームのみ公開（本名・電話番号・住所は収集しない）
- 年代・性別は任意入力・非公開
- 生徒さんへの説明：「ニックネームと投稿内容だけが他の生徒さんに見えます」
